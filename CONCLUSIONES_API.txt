================================================================================
HALLAZGOS PRINCIPALES
================================================================================

1. FORMATO DE RESPUESTA NO ESTÁNDAR
   - La API devuelve texto plano en lugar de JSON
   - Esperado: {"success": true, "token": "..."}
   - Real: "Auth_token: dGVzdHVzZXJfMTc..."
   - Solución: Parsing flexible que soporte ambos formatos

2. CÓDIGO HTTP SIEMPRE 200
   - La API devuelve 200 incluso en errores (debería ser 401, 409, etc.)
   - Solución: Validar contenido del mensaje, no solo status code

3. MENSAJES DE ERROR VARIADOS
   - "User does not exist." (usuario no registrado)
   - "Wrong password." (contraseña incorrecta)
   - "This user already exist." (duplicado)

4. TOKEN DE AUTENTICACIÓN
   - Formato: Base64 encoded string
   - Prefijo: "Auth_token: "
   - Validaciones: existencia, formato, longitud
